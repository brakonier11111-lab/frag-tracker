# 🔧 Исправление ошибки await в колбэке

## 🐛 Проблема
Ошибка: `SyntaxError: await is only valid in async functions and the top level bodies of modules`

Эта ошибка возникала из-за использования `await` внутри колбэка `db.get()`, который не был объявлен как `async`.

## ✅ Решение

### Исправление в server.js
Изменили колбэк функции `db.get()` на асинхронный:

```javascript
// Было (неправильно):
db.get('SELECT * FROM app_state WHERE id = 1', (err, state) => {
    // ...
    const stats = await getLestaPlayerStats(); // ❌ Ошибка!
    // ...
});

// Стало (правильно):
db.get('SELECT * FROM app_state WHERE id = 1', async (err, state) => {
    // ...
    try {
        const stats = await getLestaPlayerStats(); // ✅ Работает!
        if (stats) {
            console.log('✅ Синхронизация Lesta Games выполнена');
        }
    } catch (error) {
        console.error('❌ Ошибка синхронизации Lesta Games:', error);
    }
    // ...
});
```

## 🔍 Причина ошибки
- Функция `autoRefreshFragStats()` была объявлена как `async`
- Но внутри неё использовался колбэк `db.get()` без `async`
- `await` можно использовать только в `async` функциях
- Колбэк не был асинхронным, поэтому `await` вызывал ошибку

- ✅ Добавлена обработка ошибок для надежности

При использовании `await` в колбэках:
- Колбэк должен быть объявлен как `async`
- Добавляйте обработку ошибок с `try-catch`
- Убедитесь, что функция, содержащая колбэк, тоже `async`
