# 🔧 Исправление учета стартового значения в аналитике

## 🚨 **Проблема:**
Статистика донатов показывала только реальные донаты из таблицы `donations`, но не учитывала стартовое значение, которое устанавливается через кнопку "Стартовая сумма" в режиме фрагов.

**Пример:** 
- Стартовое значение: 50,000₽
- Реальные донаты: 32,356₽ (2 доната)
- Показывалось: 32,356₽ вместо 50,000₽

## ✅ **Решение:**

### **1. Обновлен API `/api/donations-stats`:**
- Теперь учитывает стартовое значение из `app_state.total_donated`
- Возвращает максимальное значение между суммой донатов и стартовым значением
- Добавлены дополнительные поля для детализации

### **2. Обновлен API `/api/donations-analytics`:**
- Корректирует общую статистику с учетом стартового значения
- Пересчитывает средний размер доната
- Обновляет минимальные и максимальные значения

### **3. Логика работы:**
```javascript
// Получаем сумму донатов из таблицы donations
const donationsAmount = Math.round(stats.totalAmount || 0);
const donationsCount = stats.totalCount || 0;

// Получаем стартовое значение из app_state
const startingAmount = Math.round(appState.total_donated || 0);

// Используем максимальное значение
const totalAmount = Math.max(donationsAmount, startingAmount);
```

## 🔧 **Техническая реализация:**

### **API Response (новый формат):**
```javascript
{
  "success": true,
  "totalAmount": 50000,        // Общая сумма (с учетом стартового значения)
  "totalCount": 2,             // Количество реальных донатов
  "donationsAmount": 32356,    // Сумма только из донатов
  "donationsCount": 2,         // Количество донатов
  "startingAmount": 50000      // Стартовое значение
}
```

### **Обработка в аналитике:**
- Если `startingAmount > donationsAmount` - корректируется статистика
- Пересчитывается средний размер доната
- Обновляются минимальные и максимальные значения

## 📊 **Результат:**
