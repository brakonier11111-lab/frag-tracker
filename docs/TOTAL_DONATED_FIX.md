# 🔧 Проблема с total_donated

## ❌ **Обнаруженная проблема:**

### 📊 **Рассинхронизация данных:**
- **`total_donated` в `app_state`** = `59856514.07` (устаревшее значение)
- **Реальная сумма из `donations`** = `59870499` (актуальное значение)
- **Разница:** `13984.93` рублей

### 🔍 **Причина проблемы:**
Значение `total_donated` в таблице `app_state` не синхронизируется с реальной суммой из таблицы `donations`. Это происходит потому, что:

1. **Донаты сохраняются** в таблицу `donations`
2. **`total_donated` обновляется** в `app_state` через функцию `processDonation`
3. **При ошибках или сбоях** значения могут рассинхронизироваться

## ✅ **Решение:**

### 🔧 **API Endpoint для исправления:**
```javascript
POST /api/fix-total-donated
```

**Что делает:**
1. Получает реальную сумму из таблицы `donations`
2. Обновляет `total_donated` в `app_state`
3. Возвращает результат операции

### 📋 **Тестовая страница:**
- **URL:** `http://localhost:3000/fix-total-donated.html`
- **Функции:**
  - Проверка текущего состояния
  - Исправление рассинхронизации
  - Отображение результатов

## 🧪 **Как исправить:**

### **Вариант 1: Через тестовую страницу**
1. Откройте `http://localhost:3000/fix-total-donated.html`
2. Нажмите "🔄 Проверить текущее состояние"

4. Проверьте результат
