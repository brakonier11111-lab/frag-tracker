# 🔧 Исправление дублирования боев

## 🐛 Проблема
Бой проходил один раз, но засчитывался дважды:
- Один бой с фрагами (от прироста фрагов)
- Один бой без фрагов (от синхронизации Lesta API)

Фраги считались корректно, но количество боев было неверным.

## ✅ Решение

### 1. Новая функция `addUniqueBattle()`
Предотвращает дублирование боев, проверяя записи за последние 5 минут:

```javascript
function addUniqueBattle(battleTime, frags = 0, source = 'lesta') {
    // Проверяем дублирование за последние 5 минут
    // Если есть записи, не добавляем новую
    // Иначе добавляем уникальный бой
}
```

### 2. Обновленная логика синхронизации Lesta Games
- **Новые бои с фрагами** → записывается один бой с фрагами
- **Новые бои без фрагов** → записывается один бой без фрагов
- **Фраги от донатов** → записываются отдельно только если нет новых боев

### 3. Скрипт очистки дубликатов
Автоматически находит и удаляет дублированные записи, оставляя запись с наибольшим количеством фрагов.

## 🚀 Изменения в коде

### Server.js
1. **Добавлена функция `addUniqueBattle()`** - предотвращает дублирование
2. **Обновлена синхронизация Lesta Games** - использует уникальные бои
3. **Обновлена обработка донатов** - использует уникальные бои

### Логика записи боев:
```javascript
// Старая логика (дублирование)
if (fragsDifference > 0) {
    addFragStats(new Date().toISOString(), fragsDifference); // Бой с фрагами
}
if (battlesDifference > 0) {
    addLestaBattle(new Date().toISOString(), 0); // Бой без фрагов

// Новая логика (уникальные бои)

    } else {
        addUniqueBattle(new Date().toISOString(), 0, 'lesta');
    }

    addUniqueBattle(new Date().toISOString(), fragsDifference, 'donation');
}
```

## 🧹 Очистка существующих дубликатов

### Запуск очистки
```bash
cleanup-duplicate-battles.bat
```

### Что делает скрипт:
1. **Анализирует все записи** в таблице `frag_stats`
2. **Группирует по времени** (с точностью до минуты)
3. **Находит дубликаты** в одной временной группе
4. **Оставляет запись с наибольшим количеством фрагов**
5. **Удаляет остальные дубликаты**
6. **Показывает результат очистки**

### Пример работы:
```
🔄 Найдены дубликаты в 2024-01-15 14:30:
   1. ID: 123, Фраги: 3, Время: 2024-01-15 14:30:15
   2. ID: 124, Фраги: 0, Время: 2024-01-15 14:30:45
   ✅ Оставляем: ID 123 с 3 фрагами
   ❌ Удаляем: ID 124 с 0 фрагами
```

## 📊 Результат исправления

### До исправления:
- Бой с 3 фрагами → 2 записи в БД
- Запись 1: 3 фрага (от прироста фрагов)
- Запись 2: 0 фрагов (от синхронизации Lesta)
- **Итого: 2 боя, 3 фрага**

### После исправления:
- Бой с 3 фрагами → 1 запись в БД
- Запись 1: 3 фрага (уникальный бой)
- **Итого: 1 бой, 3 фрага**

## 🔍 Диагностика

### Проверка дубликатов в БД
```sql
-- Найти дубликаты по времени
SELECT 
    strftime('%Y-%m-%d %H:%M', battle_time) as time_group,
    COUNT(*) as count,
    GROUP_CONCAT(id) as ids,
    GROUP_CONCAT(frags) as frags_list
FROM frag_stats 
GROUP BY strftime('%Y-%m-%d %H:%M', battle_time)
HAVING COUNT(*) > 1
ORDER BY time_group DESC;
```

### Проверка статистики
```sql
-- Общая статистика
SELECT 
    COUNT(*) as total_battles,
    SUM(frags) as total_frags,
    COUNT(CASE WHEN frags > 0 THEN 1 END) as battles_with_frags,
    COUNT(CASE WHEN frags = 0 THEN 1 END) as battles_without_frags
FROM frag_stats;
```

## 🆘 Решение проблем

### Проблема: Скрипт очистки не находит дубликаты
**Решение:**
1. Проверьте данные в БД вручную
2. Убедитесь, что есть записи с одинаковым временем
3. Запустите диагностику SQL

### Проблема: После очистки статистика неверная
**Решение:**
1. Перезапустите сервер
2. Проверьте логи сервера
3. Выполните синхронизацию Lesta Games

### Проблема: Новые бои все еще дублируются
**Решение:**
1. Убедитесь, что сервер перезапущен
2. Проверьте, что используется новая версия кода
3. Проверьте логи на наличие ошибок
