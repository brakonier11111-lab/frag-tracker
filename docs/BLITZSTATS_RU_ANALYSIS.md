# Анализ сайта BlitzStats.ru

## Обзор

**BlitzStats.ru** - это веб-сервис для отображения статистики игроков World of Tanks Blitz. Сайт предоставляет детальную статистику игроков, рейтинги, сессии и виджеты для стримеров.

**Основной URL:** https://blitzstats.ru  
**API URL:** https://api.blitzstats.ru

---

## Архитектура сайта

### 1. Технологический стек

**Frontend:**
- React (судя по структуре компонентов)
- i18next для интернационализации (локализация через JSON файлы)
- Модульная архитектура с разделением на компоненты

**Backend API:**
- REST API на `api.blitzstats.ru`
- Версионирование API (v2)

**Интеграции:**
- Lesta Games API (для получения статистики)
- Яндекс.Метрика (аналитика)
- Google Analytics (аналитика)

### 3. Страница виджетов (`/widgets`)

**URL:** https://blitzstats.ru/widgets

**Функциональность:**
- Создание виджетов для стримеров
- Требует авторизации через Lesta Games

**Причины авторизации:**
1. **Удобство** - автоматическое заполнение игровых ID
2. **Автоматизация** - старт и отсчет сессий привязан к аккаунту
3. **Контроль** - только владелец может обновлять/сбрасывать сессию
4. **Защита** - предотвращает несанкционированный доступ к сессиям
5. **Безопасность** - Lesta API не передает чувствительные данные

## API Endpoints

### 1. Получение данных пользователя

**Endpoint:** `GET https://api.blitzstats.ru/user/v2/?id={account_id}`

**Пример:**
```
GET https://api.blitzstats.ru/user/v2/?id=6098911
```

**Ответ:** JSON с данными пользователя

**Использование:**
- Загружается при открытии страницы пользователя
- Содержит основную статистику игрока

---

### 2. Получение клана пользователя

**Endpoint:** `GET https://api.blitzstats.ru/user/clan/?id={account_id}`

**Пример:**
```
GET https://api.blitzstats.ru/user/clan/?id=6098911
```

**Ответ:** JSON с информацией о клане

**Использование:**
- Отображает клан игрока в заголовке профиля

---

### 3. Получение сессий

**Endpoint:** `GET https://api.blitzstats.ru/sessions/v2/{session_id}`

**Пример:**
```
GET https://api.blitzstats.ru/sessions/v2/694a49d10c18f55d9169fc3d
```

**Ответ:** JSON с данными сессии

**Использование:**
- Загружается при открытии вкладки "Сессия"
- Содержит статистику текущей игровой сессии

---

### 4. Статус проекта

**Endpoint:** `GET https://api.blitzstats.ru/project/status`

**Примечание:** Возвращает 404, возможно не используется или устарел

## Локализация (i18n)

Сайт использует систему интернационализации с JSON файлами:

**Базовый URL:** `https://blitzstats.ru/locales/{lang}/{file}.json?v={version}`

**Файлы локализации:**
- `/locales/ru/translation.json` - основные переводы
- `/locales/ru/main.json` - главная страница
- `/locales/ru/search.json` - поиск
- `/locales/ru/nav.json` - навигация
- `/locales/ru/footer.json` - футер
- `/locales/ru/tabs.json` - вкладки
- `/locales/ru/userPage.json` - страница пользователя
- `/locales/ru/tank.json` - техника
- `/locales/ru/filter.json` - фильтры

**Версионирование:** Все файлы имеют параметр `?v={version}` для кэширования

## Особенности реализации

### 1. Отображение изменений (Delta)

Все метрики показывают изменения относительно предыдущего значения:
- **Зеленый/положительный** - увеличение (+)
- **Красный/отрицательный** - уменьшение (-)
- **Формат:** `{изменение} {текущее_значение}`

**Примеры:**
- `+667 40218 Бои` - добавлено 667 боев, всего 40218
- `-0.08% 63.74 Винрейт` - винрейт снизился на 0.08%, текущий 63.74%

### 2. Фильтрация техники

**Многоуровневая фильтрация:**
- Уровни (I-X)
- Типы (Легкий, Средний, Тяжелый, ПТ-САУ)
- Нации (9 наций)
- Премиум статус (1-4 звезды)
- Сортировка (Боёв, С/У, Винрейт, WN8, Последний бой)

**Поиск:** По названию техники

### 3. История сессий

- Хранит историю всех игровых сессий
- Формат даты: `DD.MM.YY HH:MM`
- Сортировка: от новых к старым

### 4. Изображения техники

**Источники:**
- `https://cdn.blitzstats.ru/tanks/{tank_name}.webp` - собственный CDN
- `https://glossary-na-static.gcdn.co/icons/wotb/current/uploaded/vehicles/hd_thumbnail/{tank_id}.png` - официальные изображения
- `https://wxpcdn-cbprodretail.gcdn.co/dcont/tankopedia/{nation}/{tank_id}_preview.png` - превью
- `https://ru-wotp.lesta.ru/dcont/tankopedia_images/{tank_id}/{tank_id}_image_resized.png` - Lesta CDN

## Рекомендации для улучшения вашего проекта

### 1. Добавить отображение изменений (Delta)

**Текущее состояние:**
```javascript
document.getElementById('battlesCount').textContent = currentStats.battles || '—';
```

**Рекомендуемое:**
```javascript
const previousBattles = state.lesta_previous_battles || 0;
const delta = currentStats.battles - previousBattles;
const deltaText = delta > 0 ? `+${delta}` : delta < 0 ? `${delta}` : '—';
document.getElementById('battlesCount').textContent = `${deltaText} ${currentStats.battles}`;
```

### 2. Добавить фильтрацию техники

**Реализация:**
- Получить список техники через `/api/lesta-vehicles`
- Добавить фильтры по уровням, типам, нациям
- Реализовать поиск по названию
- Добавить сортировку

### 3. Улучшить историю сессий

**Текущее:** История сохраняется в `lesta_stats_history`, но не отображается как сессии

**Рекомендуемое:**
- Группировать записи по датам
- Создать отдельную таблицу `sessions` с полями:
  - `session_id` - уникальный ID сессии
  - `start_time` - время начала сессии
  - `end_time` - время окончания сессии
  - `battles` - количество боев в сессии
  - `frags` - фраги в сессии
  - `wins` - победы
  - `losses` - поражения
  - `damage_dealt` - нанесенный урон
  - `xp` - опыт

### 4. Добавить рейтинги

**Реализация:**
- Создать таблицу `player_rankings`
- Периодически обновлять рейтинги
- Отображать топ игроков на главной странице

### 5. Улучшить API структуру

**Рекомендуемое:**
- Версионирование API (`/api/v1/`, `/api/v2/`)
- Стандартизация ответов
- Документация API (Swagger/OpenAPI)

### 6. Добавить CDN для изображений

**Реализация:**
- Кэшировать изображения техники
- Использовать собственный CDN или облачное хранилище
- Оптимизировать изображения (WebP формат)

### 7. Улучшить интернационализацию

**Текущее:** Только русский язык

**Рекомендуемое:**
- Добавить i18next или аналогичную библиотеку
- Поддержка английского языка
- Локализация через JSON файлы
