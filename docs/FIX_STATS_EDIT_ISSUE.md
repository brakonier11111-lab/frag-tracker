# 🔧 Исправление проблемы редактирования статистики

## ❌ Проблема
При редактировании статистики система изменяла общую статистику Lesta Games и показывала все бои и фраги из реальной статистики вместо локальной статистики фрагов.

## 🔍 Причина
Функции редактирования и инициализации статистики обновляли поля `lesta_last_battles` и `lesta_last_frags` в таблице `app_state`, что заставляло систему считать это общей статистикой Lesta Games вместо локальной статистики фрагов.

## 🚀 Решение

### 1. Разделение локальной и Lesta статистики
```javascript
// БЫЛО: Обновление состояния Lesta Games при редактировании
db.run(`UPDATE app_state SET 
    lesta_last_battles = ?, 
    lesta_last_frags = ?,
    lesta_previous_frags = ?
WHERE id = 1`, [totalBattles, totalFrags, totalFrags]);

// СТАЛО: НЕ обновляем состояние Lesta Games
console.log('✅ Локальная статистика фрагов обновлена');
console.log('⚠️ Состояние Lesta Games НЕ изменено - система будет отслеживать изменения от реальной статистики');
```

### 2. Добавление API для синхронизации
```javascript
app.post('/api/sync-lesta-state', async (req, res) => {

    const totalFrags = rows.reduce((sum, row) => sum + row.frags, 0);
    
    // Обновляем состояние Lesta Games только по запросу
    db.run(`UPDATE app_state SET 
        lesta_last_battles = ?, 
        lesta_last_frags = ?,
        lesta_previous_frags = ?
    WHERE id = 1`, [totalBattles, totalFrags, totalFrags]);
});
```
