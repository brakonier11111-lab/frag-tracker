# 🔄 Автообновление статистики фрагов

## 🎯 Описание
Система автоматического обновления статистики фрагов для предотвращения пропуска боев и фрагов.

## 🚀 Функции

### 1. Автоматическое обновление (каждые 30 секунд)
- **Серверная часть**: Синхронизация с Lesta Games API и DonationAlerts
- **Клиентская часть**: Обновление интерфейса статистики
- **Тихое обновление**: Без показа индикаторов загрузки

### 2. Ручное управление
- **Принудительное обновление**: Мгновенное обновление статистики
- **Запуск/остановка**: Управление автообновлением
- **Статус**: Отображение текущего состояния

## 🔧 Реализация

### Server.js
```javascript
// Функция автообновления статистики фрагов
async function autoRefreshFragStats() {
    // Проверяет Lesta Games API
    // Проверяет DonationAlerts
    // Обновляет статистику
}

// Запуск каждые 30 секунд
function startAutoRefresh() {
    autoRefreshInterval = setInterval(autoRefreshFragStats, 30000);
}
```

### API эндпоинты
- `POST /api/auto-refresh/start` - Запуск автообновления
- `POST /api/auto-refresh/stop` - Остановка автообновления
- `GET /api/auto-refresh/status` - Статус автообновления
- `POST /api/force-refresh-frag-stats` - Принудительное обновление

### Frontend (frag-stats.html)
```javascript
// Автообновление каждые 30 секунд
function startAutoRefresh() {
    autoRefreshInterval = setInterval(() => {
        loadFragStats(true); // Тихое обновление
    }, 30000);
}
```

## 📊 Логика работы

### 1. Серверное автообновление
```
Каждые 30 секунд:
├── Проверка Lesta Games API
│   ├── Получение статистики аккаунта
│   ├── Сравнение с предыдущими данными
│   └── Запись новых боев и фрагов
├── Проверка DonationAlerts
│   ├── Получение новых донатов
│   ├── Обработка донатов
│   └── Запись фрагов от донатов
└── Обновление базы данных
```

### 2. Клиентское автообновление
```
Каждые 30 секунд:
├── Запрос к API /api/frag-stats
├── Получение обновленных данных
├── Обновление интерфейса
├── Обновление графиков
└── Обновление времени последнего обновления
```

## 🎯 Преимущества

### 1. Надежность
- **Автоматическое обновление** предотвращает пропуск данных
- **Тихое обновление** не мешает пользователю
- **Обработка ошибок** при сбоях API

### 2. Удобство
- **Ручное управление** для контроля процесса
- **Визуальные индикаторы** статуса обновления
- **Время последнего обновления** для проверки актуальности

### 3. Производительность
- **Оптимизированные запросы** к API
- **Кэширование данных** для быстрого отображения
- **Минимальная нагрузка** на сервер

## 🚨 Устранение проблем

### Проблема: Автообновление не работает
**Решение:**
1. Проверьте статус через админ-панель
2. Перезапустите сервер
3. Проверьте логи на ошибки

### Проблема: Данные не обновляются
**Решение:**
1. Используйте принудительное обновление
2. Проверьте подключение к API
3. Запустите диагностику статистики

### Проблема: Высокая нагрузка на сервер
**Решение:**
1. Увеличьте интервал обновления
2. Добавьте условную логику
3. Оптимизируйте запросы к API
