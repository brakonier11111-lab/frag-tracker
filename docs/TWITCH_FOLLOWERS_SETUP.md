# 🔔 Настройка отслеживания новых фолловеров Twitch

Это руководство — для стримерского ПК, на котором реально работает `node server.js`.
Выполняйте шаги по порядку, в PowerShell/cmd на этом же компьютере.

## Зачем это нужно

Twitch должен сам «постучаться» к нашему серверу, когда кто-то зафолловил канал.
Но сервер стоит на домашнем ПК и не виден из интернета напрямую — поэтому нужен
туннель, который даёт временный публичный адрес, ведущий на наш локальный сервер.

## Шаг 1. Установить cloudflared

1. Скачать `cloudflared-windows-amd64.exe`:
   https://github.com/cloudflare/cloudflared/releases/latest
2. Переименовать скачанный файл в `cloudflared.exe` и положить, например, в
   `C:\cloudflared\cloudflared.exe`.

## Шаг 2. Запустить туннель

Открыть PowerShell в папке с `cloudflared.exe` и выполнить:

```
.\cloudflared.exe tunnel --url http://localhost:3000
```

Оставить это окно открытым — туннель работает, пока запущена эта команда.
В выводе появится блок вида:

```
+--------------------------------------------------------------------------------------------+
|  Your quick Tunnel has been created! Visit it at (it may take some time to be reachable):  |
|  https://какие-то-случайные-слова.trycloudflare.com                                        |
+--------------------------------------------------------------------------------------------+
```

Это и есть публичный адрес вашего сервера. Копируем его.

⚠️ **Важно:** при каждом перезапуске `cloudflared.exe` адрес будет **новым**.
Если это неудобно — см. раздел «Постоянный адрес» ниже.

## Шаг 3. Получить приложение на dev.twitch.tv

Если ещё не создано — на https://dev.twitch.tv/console/apps создать приложение,
взять `Client ID` и `Client Secret` (они уже должны быть в `config.env` как
`TW_CLIENT_ID`/`TW_CLIENT_SECRET` — если созданы ранее для онлайн/чата).

В настройках приложения, в поле **OAuth Redirect URLs**, добавить:
```
https://<адрес из Шага 2>/oauth/twitch/callback
```
(точь-в-точь, с `https://`, без пробелов и слеша на конце).

## Шаг 4. Прописать переменные в config.env

Открыть `config.env` в корне проекта, найти блок:
```
TW_REDIRECT_URI=https://ВАШ_ДОМЕН/oauth/twitch/callback
TW_EVENTSUB_CALLBACK_URL=https://ВАШ_ДОМЕН
TW_EVENTSUB_SECRET=придумайте_случайную_строку_10-100_символов
```
Заменить `ВАШ_ДОМЕН` на адрес из Шага 2 (без `https://` перед ним — он уже есть в шаблоне).
`TW_EVENTSUB_SECRET` — придумать любую случайную строку (10–100 символов, буквы/цифры),
никому не показывать, использовать один раз и не менять после первой настройки.

Пример итогового блока:
```
TW_REDIRECT_URI=https://college-org-weights-performances.trycloudflare.com/oauth/twitch/callback
TW_EVENTSUB_CALLBACK_URL=https://college-org-weights-performances.trycloudflare.com
TW_EVENTSUB_SECRET=x7Jk29pQmZ4wLtRb
```

## Шаг 5. Перезапустить сервер

Закрыть и заново запустить `node server.js` (или через .bat-файл, если используете),
чтобы он подхватил новые переменные из `config.env`.

## Шаг 6. Авторизовать канал

1. Открыть `http://localhost:3000/stream-integrations.html`.
2. В карточке **Twitch** найти блок «🔔 Новые фолловеры (EventSub)».
3. Нажать **«Авторизовать канал»** — откроется страница Twitch, подтвердить доступ
   (запрашивается только разрешение видеть список фолловеров, `moderator:read:followers`).
4. После редиректа обратно статус должен стать «Подключено (ваш_канал)».

## Шаг 7. Проверить

Открыть `http://localhost:3000/alert-twitch-follow.html` (можно прямо в OBS как
Browser Source) и попросить кого-нибудь зафолловить канал — должен появиться алерт
с именем нового фолловера.

## Если адрес туннеля поменялся

Если `cloudflared.exe` перезапускался и выдал новый адрес — повторить Шаг 3
(обновить Redirect URL в dev.twitch.tv) и Шаг 4 (обновить `config.env`),
затем перезапустить сервер и заново пройти Шаг 6 (повторная авторизация не помешает).

## Постоянный адрес (опционально, чтобы не переделывать при каждом перезапуске)

Вместо quick-туннеля можно создать именной туннель с бесплатным аккаунтом Cloudflare —
тогда адрес не будет меняться. Это отдельная, чуть более длинная настройка
(`cloudflared tunnel login`, `cloudflared tunnel create`, привязка к домену).
Если понадобится — попросите отдельную инструкцию под этот вариант.
