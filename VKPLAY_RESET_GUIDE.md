# Руководство по сбросу и настройке VK Play OAuth

## Шаг 1: Создание нового приложения

1. Перейдите в панель разработчика VK Play Live
2. Создайте новое приложение
3. Заполните основную информацию:
   - Название приложения
   - Описание
   - Иконка (если требуется)

## Шаг 2: Настройка OAuth

### Redirect URIs:
В настройках приложения укажите следующие URL (каждый на отдельной строке):
```
http://localhost:3000/oauth/vkplay/callback
http://localhost:3000/oauth-vkplay-implicit.html
```

**ВАЖНО:** 
- URL должны совпадать **точно**, символ в символ
- Без пробелов в начале/конце
- Без слеша в конце (если не требуется)
- Учитывайте регистр символов

### Разрешения (Scopes):
В настройках приложения выберите необходимые разрешения:
- Управление наградами за баллы канала
- Запросы наград за баллы канала
- Отправка сообщений в чат
- Управление ролями канала
- Получение баланса баллов

## Шаг 3: Получение ключей

После создания приложения вы получите:
1. **ID приложения (Client ID)** - публичный идентификатор
2. **Секретный ключ приложения (Client Secret)** - используется для OAuth CodeFlow
3. **Публичный ключ приложения (если есть)** - НЕ используется в OAuth

**ВАЖНО:** 
- Секретный ключ показывается только один раз при создании
- Сохраните его в безопасном месте
- НЕ используйте публичный ключ вместо секретного

## Шаг 4: Обновление config.env

Откройте файл `config.env` и обновите следующие строки:

```env
# VK Play OAuth
VKPLAY_CLIENT_ID=ваш_client_id_здесь
VKPLAY_CLIENT_SECRET=ваш_секретный_ключ_здесь
VKPLAY_REDIRECT_URI=http://localhost:3000/oauth/vkplay/callback
```

**Проверьте:**
- Нет лишних пробелов
- Нет кавычек вокруг значений
- Используется секретный ключ, а не публичный

## Шаг 5: Перезапуск сервера

После обновления `config.env`:
1. Остановите сервер (Ctrl+C)
2. Запустите сервер снова
3. Проверьте логи при старте - не должно быть ошибок

## Шаг 6: Тестирование авторизации

1. Откройте браузер
2. Перейдите на страницу интеграций: `http://localhost:3000/stream-integrations.html`
3. Нажмите кнопку "Подключить VK Play"
4. Проверьте логи сервера:
   - Должны быть сообщения о проверке ключей
   - Должен быть redirect_uri проверен
   - Не должно быть ошибок 401/403

## Шаг 7: Проверка после авторизации

После успешной авторизации проверьте:
1. Статус подключения: `http://localhost:3000/integrations/vkplay/status`
2. Должны быть данные:
   - `connected: true`
   - `channelUrl: "ваш_канал"`
   - `channel: {...}`
   - `tokens: {...}`

## Частые проблемы

### Ошибка 401/403 при обмене кода на токен
**Причина:** Неправильные ключи
**Решение:** 
- Проверьте, что используете секретный ключ, а не публичный
- Убедитесь, что нет лишних пробелов в config.env
- Перезапустите сервер

### Ошибка "invalid_scope"
**Причина:** Неправильные scope или они не настроены в приложении
**Решение:**
- Проверьте настройки приложения в панели разработчика
- Убедитесь, что все необходимые разрешения включены
- Попробуйте убрать scope из URL авторизации (если API не поддерживает)

### Ошибка "redirect_uri mismatch"
**Причина:** redirect_uri не совпадает с настройками приложения
**Решение:**
- Проверьте, что URL в config.env точно совпадает с настройками приложения
- Убедитесь, что нет лишних пробелов или символов
- Проверьте регистр символов

### channelUrl = null после авторизации
**Причина:** Не удалось получить данные канала
**Решение:**
- Проверьте логи сервера - должны быть запросы к API
- Убедитесь, что токен валиден
- Проверьте, что у приложения есть необходимые разрешения

## Следующие шаги

После успешной настройки OAuth:
1. Настройте автоматическую выдачу ролей при активации наград
2. Настройте отправку сообщений в чат
3. Протестируйте работу WebSocket для получения событий в реальном времени
