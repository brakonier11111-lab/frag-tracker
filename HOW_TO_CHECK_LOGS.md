# Как проверить логи сервера

## Способ 1: Консоль сервера (основной способ)

### Шаг 1: Запустите сервер
1. Откройте командную строку (PowerShell или CMD) в папке проекта
2. Запустите сервер одним из способов:
   - **Через .bat файл:** Дважды кликните на `start-server.bat`
   - **Через командную строку:** Выполните `node server.js` или `npm start`

### Шаг 2: Смотрите логи в консоли
Все логи выводятся прямо в окно консоли, где запущен сервер.

**Важные логи для диагностики VK Play:**

#### При авторизации:
```
🔐 VK Play OAuth start →
   URL: ...
   Параметры: ...
   Запрашиваемые разрешения (scope): ...
```

#### При получении наград:
```
🎁 Запрос наград VK Play:
   Channel URL: ...
   Access token: ЕСТЬ/ОТСУТСТВУЕТ
   API URL: ...
✅ Награды получены: X наград
```

#### При получении ролей:
```
📋 Запрос ролей VK Play:
   Channel URL: ...
   Access token: ЕСТЬ/ОТСУТСТВУЕТ
   API URL: ...
✅ Роли получены: X ролей
```

#### При ошибках:
```
❌ Ошибка получения наград:
   Status: 403/401/500
   Data: {...}
   Message: ...
```

## Способ 2: Консоль браузера (для ошибок фронтенда)

1. Откройте страницу управления наградами: `http://localhost:3000/vkplay-rewards.html`
2. Нажмите `F12` или `Ctrl+Shift+I` для открытия DevTools
3. Перейдите на вкладку **Console**
4. Там будут видны ошибки JavaScript и ответы API

**Что искать:**
- Ошибки загрузки наград/ролей
- Ответы от API endpoints (`/api/vkplay/rewards`, `/api/vkplay/roles`)
- Ошибки сети (Network tab)

## Способ 3: Проверка через API напрямую

Откройте в браузере:
- **Статус VK Play:** `http://localhost:3000/integrations/vkplay/status`
- **Награды:** `http://localhost:3000/api/vkplay/rewards`
- **Роли:** `http://localhost:3000/api/vkplay/roles`

Если видите ошибку `401` или `403`, проверьте логи сервера для деталей.

## Что делать при ошибках

### Ошибка 401 (Unauthorized)
- **Причина:** Токен истек или невалиден
- **Решение:** Переавторизуйтесь в VK Play

### Ошибка 403 (Forbidden)
- **Причина:** Токен не содержит нужных разрешений (scope)
- **Решение:** 
  1. Отключите VK Play
  2. Подключите заново (чтобы получить токен с новыми scope)
  3. Убедитесь, что запрашиваются все нужные scope

### Ошибка "VK Play не подключен"
- **Причина:** `channelUrl` не установлен
- **Решение:** 
  1. Нажмите кнопку "Обновить данные" на странице интеграций
  2. Или переавторизуйтесь

### Нет ошибок, но данные не загружаются
- Проверьте, что `channelUrl` установлен (в логах должно быть видно)
- Проверьте, что токен содержит нужные scope
- Проверьте логи сервера при запросе наград/ролей

## Полезные команды для проверки

### Проверить статус VK Play через curl (если установлен):
```bash
curl http://localhost:3000/integrations/vkplay/status
```

### Проверить награды:
```bash
curl http://localhost:3000/api/vkplay/rewards
```

### Проверить роли:
```bash
curl http://localhost:3000/api/vkplay/roles
```

## Сохранение логов в файл

Если нужно сохранить логи в файл:

**Windows (PowerShell):**
```powershell
node server.js | Tee-Object -FilePath "server.log"
```

**Windows (CMD):**
```cmd
node server.js > server.log 2>&1
```

Затем откройте файл `server.log` в текстовом редакторе.
