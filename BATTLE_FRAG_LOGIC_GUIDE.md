# 🎯 Логика подсчета боев и фрагов

## 🎯 Описание
Исправленная логика подсчета боев и фрагов на основе изменений в статистике Lesta Games API.

## 🔧 Принцип работы

### 1. Отслеживание изменений
- **Бои**: Сравниваем текущее количество боев с предыдущим
- **Фраги**: Сравниваем текущее количество фрагов с предыдущим
- **Разница**: Вычисляем количество новых боев и фрагов

### 2. Логика записи боев

#### Сценарий 1: Новые бои без фрагов
```
Пример: 30500 → 30501 боев, фраги не изменились
Результат: 1 бой с 0 фрагами
```

#### Сценарий 2: Новые бои с фрагами
```
Пример: 30500 → 30502 боев, +3 фрага
Результат: 
- 1-й бой: 3 фрага
- 2-й бой: 0 фрагов
```

#### Сценарий 3: Фраги от донатов
```
Пример: 30500 → 30500 боев, +2 фрага
Результат: Фраг от доната: 2 фрага
```

## 🚀 Реализация

### Server.js
```javascript
// Записываем новые бои от Lesta API с корректным распределением фрагов
const battlesDifference = stats.battles - (state.lesta_last_battles || 0);
if (battlesDifference > 0) {
    console.log(`📊 Новых боев от Lesta API: ${battlesDifference}, изменение фрагов: ${fragsDifference}`);
    
    // Распределяем фраги между новыми боями
    let remainingFrags = fragsDifference;
    
    for (let i = 0; i < battlesDifference; i++) {
        let battleFrags = 0;
        
        if (remainingFrags > 0) {
            // Если есть фраги, даем их первому бою
            battleFrags = remainingFrags;
            remainingFrags = 0;
        }
        
        // Записываем каждый бой отдельно
        addUniqueBattle(new Date().toISOString(), battleFrags, 'lesta');
        console.log(`✅ Записан бой ${i + 1}/${battlesDifference}: ${battleFrags} фрагов`);
    }
}
```

### Изменение логики распределения
```javascript
// Равномерное распределение фрагов
if (remainingFrags > 0) {
    battleFrags = Math.floor(remainingFrags / (battlesDifference - i));
    remainingFrags -= battleFrags;
}

// Распределение по приоритету
if (remainingFrags > 0 && i === 0) {
    battleFrags = remainingFrags;
    remainingFrags = 0;
}
```
